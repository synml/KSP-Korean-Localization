# 번역 전수 검토 체크리스트

KSP 한국어 번역(dictionary.ko.cfg + BreakingGround.ko.cfg)을 검토할 때의 점검 항목.
위에서 아래로 갈수록 자동화가 어렵고 사람 판단이 필요하다. **기계 검증(1)을 먼저 통과시킨 뒤 의미·문체(5~6)를 본다.**

- 유일 기준: [glossary.md](glossary.md)(용어)와 [CLAUDE.md](../CLAUDE.md)(규칙·의도적 유지 목록).
- 수정은 항상 **GameData 쪽 파일 원문을 재확인한 뒤** 적용한다(대조 인용 시 무의식적 "교정" 오탐 주의).
- 대량 수정은 검증 가능한 스크립트로 하고, 매 단계 `validate.py` 오류 0을 유지한다.

---

## 1. 기계 검증 (커밋 전 필수 · `scripts/validate.py`)

- [ ] **키 집합 1:1 일치** — reference_english ↔ ko. 누락 키는 영어 fallback이 아니라 **키 원문**이 노출되므로 치명적.
- [ ] **Lingoona 변수 태그 보존** — `<<1>>`, `<<A:1>>`, `<<n:1[a/b/c]>>` 집합이 원문과 동일. 삭제·변형 금지(어순 이동은 허용).
- [ ] **리치 텍스트 태그 짝** — `<b></b>`, `<color=#...></color>`, `<i>` 등 여닫음 균형.
- [ ] **값 내 금지 문자** — 실제 개행 없음, 값 안 `//`(주석으로 잘림), 짝 없는 `{`·`}`.
- [ ] **cfg 문법** — 중괄호 균형, `노드 { ko { ... } }` 구조.
- [ ] **파일 규격** — UTF-8 BOM + CRLF + 탭 들여쓰기.

> 경고(warning)는 원문 자체 결함을 재현한 것일 수 있다 → CLAUDE.md '영어 원문 결함 목록'과 대조해 **오류가 아님**을 확인.

## 2. 태그·서식 세부 (validate가 개수만 보는 부분)

- [ ] `<<n:1[형태a/형태b]>>` 대괄호 **안**의 한국어 형태가 자연스러운가(개수만 검증됨).
- [ ] 빈 형태 2개 연속(` / / `)이 `//`로 잘리지 않게 공백을 채웠는가.
- [ ] `\n`(문자 그대로 백슬래시+n) 개수·위치가 의도대로인가.
- [ ] 색상·볼드 범위가 의도한 텍스트만 감싸는가(닫는 태그 위치).

## 3. 용어 일관성 (glossary.md 기준)

- [ ] 표준 용어로 통일됐는가 — 예: 메뉴버·스로틀·우주선·관제소·경유지(Waypoint)·방열기(Radiator)/방열판(Heat Shield)·커맨(Kerman).
- [ ] 커뮤니티 통용 영어를 억지 한글화하지 않았는가(Delta-v, EVA, RCS 등).
- [ ] 파트 별명 음차·따옴표가 원문 방식을 따르는가(엔진 `"`, KV 귀환선 `'` — 임의 통일 금지).
- [ ] 천체·용어 표기 — 커빈(≠지구), 태양/태양계(≠커볼/커볼계), 근접 통과(Flyby).

## 4. 문법 · 맞춤법 · 조사

- [ ] **조사-받침 일치** — 을/를·은/는·이/가·과/와·(으)로. 받침 있으면 을/은/이/과/으로, 없으면 를/는/가/와/로(ㄹ받침은 '로').
- [ ] **용어 일괄 치환의 조사 부작용** — 표준 용어로 바꾸면 앞말 받침이 달라져 뒤 조사가 깨진다(예: 치환 뒤 '격납고을' → '격납고를'). 3절 용어 통일 후 **바뀐 단어 바로 뒤의 조사**를 전수 재확인한다.
- [ ] **가변 태그 뒤 조사** — `<<1>>` 등 값이 런타임 결정이면 `(이)가`·`(을)를`·`(으)로` 관용 표기(형태를 하나로 확정하지 말 것). 괄호 위치 주의: `(으)로` (○) / `으(로)` (✗).
- [ ] **조사는 앞말에 붙여쓴다** — 처럼·밖에·부터·까지·마다·조차·뿐(체언 뒤)·만·보다. (단 관형형 뒤 `뿐`·`데`·`수`는 의존명사 → 띄어씀.)
- [ ] **로서(자격/신분) ↔ 로써(수단/도구)** — "엔진으로서"(○ 자격) / "함으로써"(○ 수단).
- [ ] **되/돼** — '돼'=되어. "돼서/됐다/돼야"(○), "되서/됬다/되야"(✗).
- [ ] **율/률** — 모음·ㄴ받침 뒤 '율'(비율), 그 외 받침 뒤 '률'(확률·손실률·발생률).
- [ ] **의존명사 띄어쓰기** — 수·것·때·데·개·명·동안·뿐·바 등은 앞말과 띄움("할 수 있다", "몇 개", "했을 때").
- [ ] **명사+하다 붙여쓰기** — "사용하다·통과하다·불가능하다"(붙임), "삭제하시겠습니까".
- [ ] **흔한 오타** — 어차피·때문·뵙도록·생각지도·곰곰이·웬만하면 등, 이중 공백, 조합 낱자(ㄱ·ㅏ) 노출(단 의도적 깨진 텍스트는 예외).

## 5. 문체 · 레지스터

- [ ] **유형별 문체 통일** — 파트 설명 "~합니다", 튜토리얼 "~하세요".
- [ ] **과학 리포트는 평서체(일지 어조)** 유지 — 1인칭 관찰·측정 로그는 파트 설명체로 바꾸지 않는다(오너 결정).
- [ ] **유머·말장난** — 직역하지 말고 정보(스펙/경고)만 보존해 재창작 허용. 단 스펙 수치·경고는 정확히.
- [ ] **UI 라벨 압축** — 버튼·표 헤더는 잘림 방지를 위해 최대한 짧게.

## 6. 의미 정확성 (reference_english 1:1 대조)

- [ ] 원문 의미가 왜곡·누락되지 않았는가(특히 수치·단위·방향·조건).
- [ ] 부정/조건/비교(not, unless, than) 논리가 뒤집히지 않았는가.
- [ ] 고유명(크루명·기체명·미션명) 처리 방침이 일관되는가(실명 이스터에그 → 음차 통일).
- [ ] 본체·BG **공통 키**(8014149~8014151 등)는 두 파일에 동일하게 반영했는가.
- [ ] **판단이 갈리는 의미 교정은 즉시 확정하지 말 것** — '오역으로 보이는' 값도 원문 의도가 옳을 수 있다(교정 후 오너가 원래 값으로 롤백한 선례 다수). 확신이 없으면 pending 목록으로 넘겨 오너 결정을 받는다.

## 7. 의도적 유지 항목 (건드리지 말 것)

- [ ] **ko==en 잔여**는 대부분 의도적 유지 — `scripts/untranslated.py`로 분류 확인.
  - keyword(파트 검색 태그·소문자)·symbol(단위/기호/모델명)·일부 todo(고유명·내부 식별자)는 **번역 금지**.
- [ ] **원문 결함 재현** — CLAUDE.md 목록의 태그 오타(`<<1>`, `<\color>`)·철자 오타는 그대로 둔다(validate 통과 조건).
- [ ] **의도적으로 깨진 텍스트** — 예: #autoLOC_501435(깨진 통신문). 조합 낱자·기호가 원문 대응이면 유지.
- [ ] **제품명·확장팩명** 등 영어 유지 결정 항목(Making History 등).

## 8. 인게임 최종 확인 (스크립트로 못 잡는 것)

- [ ] UI 라벨 **잘림/줄바꿈** — 버튼·툴팁·표에서 텍스트가 넘치지 않는가.
- [ ] 폰트 렌더링 — 희귀 음절(뷁 등)·IME 조합 낱자가 □로 깨지지 않는가.
- [ ] 가변 태그 실제 치환 결과 — `(이)가` 등이 실행 시 자연스러운가.
- [ ] 색상·볼드가 의도대로 보이는가.
