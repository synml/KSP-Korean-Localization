# 인게임 확인 체크포인트

스크립트(validate.py)로 못 잡는 것 — **UI 잘림·폰트 렌더링·가변 태그 실제 치환·색상/볼드** — 을 실제 게임에서 확인하는 목록.
[review-checklist.md](review-checklist.md) 8절의 실전 확장판. 화면별로 이동해 각 항목을 눈으로 확인한다.

- 기준: 언어가 `ko`로 전환된 상태(플러그인 자동). 폰트는 `korean`/`korean2` 번들 로드 확인.
- 표기: `[ ]` 미확인 → 확인 후 체크. 문제 발견 시 키(#autoLOC_…)를 메모해 GameData 파일에서 수정.
- **★ = 이번 검토(26.07.21)에서 수정한 항목** — 실제 화면에서 반영·자연스러움을 우선 확인.

---

## 0. 부팅 · 폰트 스모크

- [ ] 메인 메뉴 진입까지 □(두부) 없이 한글이 정상 렌더링.
- [ ] **희귀 음절 렌더링** — 기체/저장 파일 이름에 `뷁·뎳·꿹` 등 완성형 밖 음절 입력 시 □ 안 뜸(korean2 아틀라스, DX10.1+ GPU 필요).
- [ ] **IME 조합 낱자** — 텍스트 입력칸(기체명·깃발·저장명)에서 한글 조합 중 낱자(ㄱ·ㅏ·ㅘ)가 □로 안 깨짐.
- [ ] 폰트 크기·자간이 라틴과 섞일 때(예: `Δv`, `Mk1`) 어색하게 겹치지 않음.

## 1. 메인 메뉴 · 설정

- [ ] 메뉴 버튼(새 게임/불러오기/설정/나가기) 라벨 잘림 없음.
- [ ] 설정 화면 탭·라벨(그래픽·소리·입력) 잘림/줄바꿈 확인.
- [ ] **날짜 표기 토글** — 설정의 "커빈력으로 표시(1일:6시간, 1년:426일)"/"지구력으로 표시(1일:24시간, 1년:365일)" 수치·문구 정상.
- [ ] 키 바인딩 목록 라벨 잘림 확인(항목 많음).

## 2. VAB / SPH (조립동)

- [ ] 부품 카테고리 탭(조종부·연료·엔진·구조물…) 라벨 잘림 없음.
- [ ] 부품 정보창(제목·설명·스펙) 스크롤/잘림 확인.
- [ ] ★ **Mk3 화물실 설명** — CRG-50 설명에 "X200-32 연료통", CRG-25 설명에 "X200-16 연료통", CRG-100 설명에 "점보-64 연료통"이 각각 맞게 나오는지(과거 셋 다 점보-64였음).
- [ ] ★ **FTX-2** 부품 플레이버가 "연료관"으로(‘연료 덕트’ 아님).
- [ ] 파트 별명 따옴표 — 엔진 `"섬퍼"·"매머드"`(큰따옴표), KV 귀환선 `'어니언'·'피'`(작은따옴표) 원문 방식 유지.
- [ ] 액션 그룹 편집 UI — "조향장치·제동장치·조명장치" 등 라벨, 액션 그룹 이름 잘림.
- [ ] 부품 우클릭(PAW) 슬라이더/토글 — ★ 방향 설정 드롭다운에 "일반 / 역방향 / **반전**"이 서로 구분되어 표시(과거 역방향 중복).
- [ ] 승무원 배치창 — 이름/역할(조종사·기술자·과학자·관광객) 표기.

## 3. 발사 · 비행 (Flight)

- [ ] **네브볼(항법구)** SAS 마커 라벨 — 순행/역행/법선/반법선/구심/원심(방향) 잘림 없음.
- [ ] 상단 시간/자원 게이지 라벨(전력·단일추진제·액체연료·산화제) 잘림.
- [ ] ★ **시간 가속** 표시 — 가속/감속 시 문구가 "시간 가속/시간 감속"(‘워프’ 음차 아님).
- [ ] 스테이징 목록·연소시간·`Δv` 표시(‘델타-v/델타V’ 음차 아님, `Δv`로).
- [ ] 고도/속도/`Δv` 판넬 수치+단위(`m/s`, `m²`, `km`) 정렬·잘림.
- [ ] 경고/알림 토스트(빨강·주황) — 색상·볼드가 의도 텍스트만 감싸는지(예: 공력 파손·정박 잠김 메시지).
- [ ] 상황 표시(착륙함·궤도 선회중·준궤도 비행중 등) 표기.
- [ ] EVA 버튼·컨텍스트 메뉴 라벨.

## 4. 지도 보기 · 궤도 (Map View)

- [ ] 궤도 정보 라벨 — 최원점(Ap)/최근점(Pe)/경사각/영향권(SOI) 잘림.
- [ ] **메뉴버 노드** 편집 핸들·`Δv`·연소시간 표시.
- [ ] 근접/조우 마커, 근접 통과(Flyby) 표기.
- [ ] 천체명 음차(문/민무스/듀나/줄…) + 성별 태그(`^N`) 렌더 정상.

## 5. 튜토리얼 (문체·가변 태그 집중)

- [ ] 문체가 "~하세요"체로 일관.
- [ ] ★ **도킹 제어 모드** — 도킹 튜토리얼에서 "도킹 제어 모드"/"스테이지(스테이징) 제어 모드"로(‘컨트롤/조작’ 아님).
- [ ] ★ **승교점/강교점** — AN/DN 노드 튜토리얼 **제목**이 "승교점(AN), 강교점(DN) 노드", 본문도 동일 용어.
- [ ] 가변 태그 실제 치환 — `<<1>>(이)가`·`(을)를`·`(으)로`가 천체·기체 이름 대입 후 자연스러운지(예: "문이", "민무스가").
- [ ] 색상/볼드 강조(`[스페이스]` 등 키 안내)가 의도대로.
- [ ] 오타 스팟 — "헤매다가"(‘헤메’ 아님), "~을 겁니다"/"~을 텐데"(붙여쓰기 아님).

## 6. 과학 · 커리어

- [ ] 과학 리포트 블록 문체 — 평서체(일지 어조) 유지, 파트 설명체로 안 바뀜.
- [ ] 실험/시료 용어 — "지표면 시료"·"승무원 보고서"·"EVA 보고서", 시료↔샘플 혼용 없음.
- [ ] 계약 목록·본문 — 가변 태그(`[Agency]`·`[Topic]` 대입) 문장 자연스러움, "(은)는/(이)가" 병기 렌더.
- [ ] R&D 기술 트리 노드·과학점수 표기, 자금/평판 UI.
- [ ] 회수/전송 완료 토스트 — `<<1>>(이)가 무사히 회수…` 실제 이름 치환 확인.

## 7. 미션 빌더 (MB) · Making History

- [ ] 노드 팔레트·액션 창(설정 액션 창/그래픽 액션 창) 라벨 잘림.
- [ ] ★ **거리 계산 노드** — 유형 선택기 두 개가 "거리를 계산할 **기준** 대상…"(from) / "…**목표** 대상…"(to)로 구분됨.
- [ ] 미션/임무 구분 — MB 기능은 "미션", 일반 과업은 "임무" 문맥 확인.
- [ ] 대사 노드 색상/볼드(`<b><i>…`) 렌더, 시나리오 기체 고유명(영어 유지) 표시.

## 8. Breaking Ground (DLC)

- [ ] 로보틱스 파트(힌지·피스톤·로터·회전 서보) 라벨·PAW.
- [ ] 전개형 과학장비(Deployed Science) — 실험 설명·전력 요구·통신 범위 수치.
- [ ] 지표면 특징(Surface Features) 스캔 — ★ "일루 빙산"(‘버그’ 아님) 등 명칭·설명.
- [ ] 그립 패드·전개/수납 라벨.

4. 튜토리얼 — "도킹 제어 모드", "승교점/강교점 노드".
5. MB 거리 계산 — "기준 대상 / 목표 대상".
